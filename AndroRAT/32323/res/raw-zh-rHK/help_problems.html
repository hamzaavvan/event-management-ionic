分割區沒有足夠的空間

請嘗試在設定中更改目標 hosts 檔為 /data/data/hosts 及再次套用到 AdAway。

AdAway 未能在某些應用程式中攔截廣告！

有些主機名稱可能在「Hosts 來源」所提供的 hosts 檔中沒有收錄，又或者應用程式本身已捆綁內建的廣告圖像，所以應用程式就算無連接到網絡也可顯示廣告。

您可使用 AdAway 的記錄 DNS 請求功能 (選單 -> 記錄 DNS 請求) 去尋找更多可以攔截的主機名稱。

只要在記錄中，長按那些可疑的主機名稱就可加至您的黑名單內。同時您可到 hosts-file.net 中的論壇子分區 Hosts Inbox 頁面去驗證那些可疑的主機名稱，如果驗證屬實同時可在該區回報。

儲存於快取中的廣告

某些應用程式會在下載完成後，同時將廣告儲存於快取中，這導致 AdAway 失效及令到廣告依然會顯示於這些應用程式。您可嘗試在 Android 的管理應用程式中，選擇清除相關應用程式的快取去繞過這問題。

當使用 AdAway 及開啟廣告攔截後，某些應用程式就不能正常運作甚至無法運作！

某些應用程式必須和被 AdAway 攔截的廣告主機名稱進行溝通，強行攔截會令這些應用程式不能正常運作甚至無法運作。請觀看 AdAway 於 Google Code 專案中的有問題應用程式頁面以獲取有問題應用程式的名單。另外，您可自己查明及加入那些特定的主機名稱到「您的名單」中的「白名單」內，同時可在 AdAway 於 Google Code 專案中的臭蟲追蹤頁面進行回報。
